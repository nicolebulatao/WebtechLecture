Fundamentals

HTTP is a stateless communication protocol that runs on top of the TCP/IP.

HTTP using TCP uses port 80 by default

HTTPS using TCP uses port 443 by default

  • Scheme
  • Authority—user information and authentication
  • Host must be identified
  • Port number
  • Resource Path – Includes actual document or resources
  • scheme specific parameters(optional)
  • URL-encoded query(optional)
  • Bookmark (fragmented identifier) – subjection of a specified document(optional)

    • Cookie (web cookie, browser cookie) -refer to a small piece of data that sends by the server to the user through the browser.

    Cache -Local storage of resources that is fetched from a server
